What is ACID property of database?

ACID (atomicity, consistency, isolation, durability) is a set of properties of database transactions intended to guarantee data validity despite errors, power failures, and other mishaps

What is a transaction in database?

Sequence of database operations that satisfies the ACID properties (which can be perceived as a single logical operation on the data) is called a transaction.

Example - Withdrawing money from ATM is a transaction.

1. Customer account is debited

2. Customer has received the money from ATM machine

Either both should succeed or none should succeed to ensure the database transaction is ACID compliant

What is a Clustered index?

With a clustered index the rows are stored physically on the disk in the same order as the index.

Therefore, there can be only one clustered index. Why? As items can be arranged in only one specific order at a given point of time, there can be only one clustered index

It is generally faster to read from a clustered index if you want to get back all the columns.

You do not have to go first to the index and then to the table.

Writing to a table with a clustered index can be slower, if there is a need to rearrange the data.

What is a non-clustered index?

With a non clustered index there is a second list that has pointers to the physical rows. You can have many non clustered indices, although each new index will increase the time it takes to write new records.

What is the advantage of Clustered index?

Reading data from the table would be faster if the SELECT query has WHERE condition based on the column/columns through which clustered index is created

What is the disadvantage of Clustered index?

Writing to a table with a clustered index can be slower, if there is a need to rearrange the data, on insert/update/delete of data.
